Introduction

Format and syntax of the name components shall be as shown in the figure below. Only the device name is required on drawings, name tags, etc. where the drawing or device name clearly indicates the system and subsystem including the equipment. However, where the system and/or subsystem are not apparent, the full name must be shown.

Examples

Description Name Comment SOFBDev is a CDev belonging to system SI, subsystem CO, whose function is to control the slow-orbit feedback system. 'Mode' is a state variable of the CDev that can be set or read to turn correction on|off. SI-Soft:CO-SOFB:Mode TxDev: - RxDev: - The property excitation amplitude ExcAmp of the beam shaker has a RxDev the shaker itself (TuneShk-17SA) and, as TxDev, the TuneDev generating the shaker signal. The choice of picking TuneShk, the RxDev for the Dev subfield, is in accordance with R08b for, the other option, TxDev TuneDev would convey less functional relevance. SI-17SA:DI-TuneShk:ExcAmp TxDev: SI-DI:TuneDev RxDev: SI-DI:TuneShk The other option for Dev here could be RxDev TuneDev. But the TxDev TunePickUp is more relevant since, apart from indirectly specifying the locating of the origin of the signal, the TxDev name is more closely associated with the signal relevance. Besides, if TuneDev were chosen and another tune pickup added to the storage ring later, the new PV name could not follow the same rationale as the previous existing one, as the new PV name should need to differentiate it from the previoius existing one by addng the info of the new pickup either in its Dev or in its Prop subfields. SI-01SB:DI-TunePkp:TuneX TxDev: SI-DI:TunePkp RxDev: SI-DI:TuneDev The TxDev is the Dev BPM-01M2 and the RXDev is the BPM electronics BPMElct-01 (that will process other BPM signals as well). BPM-01M2 is the particular whereas BPMElect-01 is more general since this electronics may process more BPM signals. SI-01M2:DI-BPM:MonitPosX or SI-01M2:DI-BPM:MonitPos-X TxDev: SI-DI:BPM-01M2 RxDev: SI-DI:BPMElct-01
